### Q.

- 개인적으로 하는 프로젝트에서 사용자들간의 1:1 채팅 기능을 구현하려고 하는데 이때 채팅룸과 채팅 메시지 등에 대한 정보를 DB에 저장하려고 합니다.

- 이 경우에는 sql과 nosql중 어떤걸 사용하는게 좋은지 만약 nosql이라면 어떤 종류의 nosql(mongoDB, 카산드라 등)이 추천되는지 혹시 가르쳐주실 수 있을까요?

- 채팅 기능 같은 경우는 흔히 볼 수 있는 기능이니까 이 강의를 보는 다른 분들께도 도움이 될 거 같습니다.

### A.

- 사용자 + 채팅과 같은 관계가 있는 데이터에는 거의 대부분의 경우에는 SQL이 NoSQL보다 더 선호될 것 같아요. 보통의 use cases (기능들)이 다 커버가 되기 때문이죠.

- 그리고 채팅 어플리케이션이라고 해도, 어떤 기능이 우선시 되는지에 따라서 다를것 같아요 :) 심플한 채팅 기능만 있는 경우라면 NoSQL로도 충분히 구현 가능하지만, 제가 챕터 20부터 드위터 프로젝트에 구현하면서 언급하는 내용들도 참고해서 SQL vs NoSQL 선택의 각기 다른 장/단점에 대해 이해하고 고르셔야 할 것 같아요 :)

- 챕터 20부터 구현하면서 중간 중간에 제가 언급하는 내용들 확인해 보시면 도움이 되실 것 같아요 ;)

### Q.

- SQL vs NoSQL의 세번째 항목 'How data will be queried' 부분에서 제 이해가 맞는지 확인하고 싶습니다!

- NoSQL

  - 개발하려는 서비스에서 테이블 간 관계가 복잡하지 않은 경우

  - 단순 조회(SELECT) 연산으로 대부분의 기능을 제공할 수 있는 경우

- SQL

  - 개발하려는 서비스에서 테이블(또는 엔티티) 간 합쳐지는 쿼리(JOIN)가 많이 발생하는 경우

  - 단순 조회 연산 외에도 갱신(추가, 수정, 삭제) 작업이 많이 필요한 경우

### A.

- 넵, 정리 정말 잘하셨어요 💯
